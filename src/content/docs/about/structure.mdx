## Committee Model

The FCS project is divided into four functional committees. Each committee owns a distinct engineering domain and is responsible for design, build, test, and documentation within that scope.

## Committee List

| Committee | Scope |
| --- | --- |
| Controls and Dynamics | PID control, sensor fusion, state estimation, flight algorithms |
| Hardware and Fabrication | Airframe, motors, propellers, mechanical assembly |
| Electronics and Power | ESCs, power distribution, wiring, connectors, PCBs |
| Simulation and Software | Simulation, ground station tools, data logging |

## Why this structure works

- It gives each team a clear technical boundary
- It still encourages cross-functional collaboration
- It creates accountable documentation ownership
- It maps cleanly onto real engineering workflows

## Documentation expectations

Each committee should maintain:

- Design decisions and rationale
- Build or implementation notes
- Test procedures
- Validation results
- Open issues and next steps
